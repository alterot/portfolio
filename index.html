about

Is it possible to become a Full Stack Web Developer while having a full-time job and two kids? That’s what I was hoping early in 2022 when I decided to plunge headfirst into the rabbit hole of learning JavaScript on my own. Before then, I hadn't done much in terms of web development since I was a kid, and the <frame> attribute was super hot, so I wasn’t sure what to expect, but it has truly been a blast and the online community is great and super helpful!

Admittedly, it’s not always easy, but every time my motivation or confidence levels sink to miserable levels, I look back at all I've learned in this past year and a half and that really keeps me going!

portfolio

Here are some of my previous projects that I've created purely for my own amusement or to enhance my understanding of a specific aspect of JavaScript.

Also, down in the bottom left corner you'll find a link to my GitHub profile if you're interested in reviewing some of the code or exploring more of my work.

Popular designs from Dribble.com

Preparing for a future as a web developer, I aimed to replicate a few of the most popular designs on Dribble.com as if they were mockups from a designer. I didn’t have access to exact fonts, pixels, images, etc., but I strived to stay as true to the original as possible (changing a font or adding an image later down the line isn’t super hard after all).

I have consistently used BEM methodology in my styling, and to present the designs I created an image slider with TypeScript and React. They are optimized for desktop.

Spellbook library TypeScript

I created a new version of my spellbook library. This time using TypeScript and BEM methodology for the styling. I set up an asynchronous integration with fetch() to load a local JSON file with all spell data and display the relevant spells in accordance with selected filters.

The code is modular and easily maintainable (I hope) with separate functions to handle user interactions in a clear and structured way.

Website for a local cocktail bar

I developed a simple dynamic website for a local cocktail bar (very local, it is in fact, located on my porch).
I developed the application using Webpack and a Single Page Application architecture to dynamically load content without full page reloads. Each page’s functionality and styling is encapsulated into separate modules, enhancing code organization and maintainability.

Black Sabbath random GIFs

I set up an integration with an open API (giphy.com) that will fetch a random Black Sabbath GIF when you load the page.

In hindsight it might not have been very clever to fetch and display a random GIF on your portfolio site. Especially considering the (awesome) nature of the band in question, but so far it's gone ok. No GIFs on eaten bats thank god.

Spellbook library

I created a spellbook library for my ttrpg group, providing functionality to add, remove, and display spells.
I used object-oriented principles to create the constructor function with relevant properties, and event listeners to handle form submissions when adding new spells and remove buttons to delete spells from the spell list. I implemented a Local Storage API to store and retrieve data in JSON format.

Tic-Tac-Toe

PvP Tic-Tac-Toe game I created with a Mobile First design for the CSS layout.
It is primarily function-oriented, but I also wanted to demonstrate the use of object-oriented concepts (factory function and module pattern) to encapsulate data and logic related to players and the game board. The game logic handles win-checking and draw scenarios.

Etch-a-Scetch

A straightforward Etch-a-Sketch with different color modes and grid sizes.The purpose was primarily to improve my event handling and DOM manipulation skills. Currently based on mouse interaction, but if there is a strong demand (which is surely very likely) I'll add touch support as well to enable use on mobile devices.
